> 扩展 2.6.5 节中的词法分析器，使它能够识别关系运算符 `<`、`<=`、`==`、`!=`、`>=`、`>`。

还是类似之前的，读到首个字符后下一个字符不是 `=` 不能返回失败，而是应当返回第一个字符单独的 `Token`。主要是 `=` 和 `!` 两个。

为了隔离开不同题目的实现，代码没有和上一题放到一起，实际上是可以合并的，因为两题涉及的字符交集为空。

这次其实不需要用 `java.io.PushbackReader`，直接从上一题复制了代码，没有改初始化这块。

参考的测试用例：

```
<=<!====!>=>
```

预期输出是

```
<rel "<=">
<rel "<">
<rel !=>
<rel ==>
<token =>
<token !>
<rel ">=">
<rel ">">
```
